Hexo建立个人博客-Ubuntu

Hexo建立个人博客-Ubuntu

什么是hexo?

hexo是一个基于Node.js的静态博客程序，可以方便的生成静态网页托管在github和Heroku上。类似于jekyll、Octopress、Wordpress，我们可以用hexo创建自己的博客，托管到github或Heroku上，绑定自己的域名，用markdown写文章。
hexo的优点:

  • 不可思议的快速，只要一眨眼静态文件即生成完成
  • 支持 Markdown
  • 仅需一道指令即可部署到 GitHub ，Pages 和 Heroku
  • 已移植 Octopress 插件
  • 高扩展性、自订性
  • 兼容于 Windows, Mac & Linux
  • 平时使用中仅需要hexo new hexo generate hexo server hexo deploy四个命令，不像Jekyll需要很多繁琐的git命令

如何托管到Github?

在Github主页有下角创建一个repository，命名为：zichun-h.github.io,在本地blog文件夹中打开_config.yml文件，修改以下部分:

deploy:
type: git
repo: https://github.com/zichun-h/zichun-h.github.io.git
branch: master

注意： 新版本的 hexo 要把 type 写成 git 而不是 github；同时因为ymal规范的原因，必须使用空格缩进，并确认冒号后面要添加一个空格。

问题

  1. 在github部署完成之后，马上访问可能出现404错误，这是正常的，（最多）等待十分钟左右就可以访问了。如果还不行，那很可能是 github 发送给你的验证邮件你没有打开看，据多方反映，验证后就没问题了。
  2. 出现乱码的，不要使用 windows 中的「记事本」打开并编辑文件，推荐使用 sublime text，很简单。如果已经在「记事本」中编辑过，需要使用 sublime text 转码为「utf8」。
